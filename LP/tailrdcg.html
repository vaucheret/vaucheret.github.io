Tail Recursion

Recursivo a la Cola

Tail Recursion

  • Una relacion recursiva es tail recursive solo si:

    • hay una sola llamada recursiva
    • la llamada recursiva es la última meta de la última cláusula
    • todas las metas precediendo la última meta en la última cláusula son deterministas
  • En Prolog, las relaciones tail recursive pueden ser optimizadas por el sistema de ejecución para evitar el crecimiento de la pila de llamadas.
  • Cuando Prolog detecta que una relación es tail recursive, transforma la llamada recursiva en una iteración.

Ejemplo de Tail Recursion

Sumatoria de una lista de números:


sum([], 0).
sum([H|T], S) :-
    sum(T, S1),
    S is H + S1.

Ejemplo de Tail Recursion Optimizada

Sumatoria de una lista de números utilizando un acumulador:


sum(List, S) :-
    sum_helper(List, 0, S).

sum_helper([], Acc, Acc).
sum_helper([H|T], Acc, S) :-
    Acc1 is Acc + H,
    sum_helper(T, Acc1, S).

Ejemplo de Tail Recursion


factorial(0, 1).

factorial(N, F) :-
    N > 0,
    N1 is N - 1,
    factorial(N1, F1),
    F is N * F1.

Ejemplo de Tail Recursion Optimizada


factorial(N, F) :-
    factorial_helper(N, 1, F).

factorial_helper(0, Acc, Acc).
factorial_helper(N, Acc, F) :-
    N > 0,
    Acc1 is Acc * N,
    N1 is N - 1,
    factorial_helper(N1, Acc1, F).

Ultimo Ejemplo: Fibonacci


fibonacci(0, 0).
fibonacci(1, 1).

fibonacci(N, F) :-
    N > 1,
    N1 is N - 1,
    N2 is N - 2,
    fibonacci(N1, F1),
    fibonacci(N2, F2),
    F is F1 + F2.

Ejemplo de Tail Recursion Optimizada


fibonacci(N, F) :-
    fibonacci_helper(N, 0, 1, F).

fibonacci_helper(0, A, _, A).
fibonacci_helper(N, A, B, F) :-
    N > 0,
    A1 is B,
    B1 is A + B,
    N1 is N - 1,
    fibonacci_helper(N1, A1, B1, F).

Ventajas de Tail Recursion

  • Reduce el consumo de memoria al evitar el crecimiento de la pila de llamadas.
  • Mejora el rendimiento al transformar la recursión en iteración.
  • Permite resolver problemas que de otra manera podrían causar un desbordamiento de pila.

Invertir una lista utilizando Tail Recursion

Invertir una lista, version recursiva:


reverse([], []).
reverse([H|T], R) :-
    reverse(T, RT),
    append(RT, [H], R).

Invertir una lista, version tail recursive:


rev_list(L, R) :-
    rev_listAux(L, [], R).

rev_listAux([], R, R).
rev_listAux([H|T], A, R) :-
    rev_listAux(T, [H|A], R).

Diferencia de Listas

La diferencia de listas es una técnica utilizada en Prolog para representar listas con punteros a la cabeza y la cola de la lista. Esto permite realizar operaciones de concatenación y manipulación de listas de manera eficiente.

Lista [a,b,c] como diferencia de listas

La lista [a,b,c] se puede representar como una diferencia de listas de la siguiente manera:

  • [a,b,c,d,e] - [d,e]
  • [a,b,c] - []
  • [a,b,c,1] - [1]
  • [a,b,c|T] - T

La lista vacia como diferencia de listas

La lista vacía se puede representar como una diferencia de listas de la siguiente manera

  • [] - []
  • [a,b,c] - [a,b,c]
  • L - L

add_front/3 es un predicado que agrega un elemento al frente de una lista.


add_front(X, L, [X|L]).

Agregar un elemento a una lista

add_back/3 es un predicado que agrega un elemento al final de una lista recorriendo la lista hasta el final.


add_back(X, [], [X]).
add_back(X, [H|T], [H|R]) :-
    add_back(X, T, R).

add_back/3 es menos eficiente que add_front/3 debido a que recorre toda la lista para agregar el elemento al final, mientras que add_front/3 simplemente agrega el elemento al frente de la lista.

Agregar un elemento a una lista

La diferencia de listas permite agregar elementos al final de una lista de manera eficiente sin necesidad de recorrer toda la lista.

add_back_dif/3 es un predicado que agrega un elemento al final de una lista utilizando la diferencia de listas.


add_back_dif(X, L - [X|R], L - R).

Concatenar dos listas

La concatenación de dos listas recorriendo la lista hasta el final se puede realizar de la siguiente manera:


concatenate([], L, L).
concatenate([H|T], L2, [H|R]) :-
    concatenate(T, L2, R).

Concatenar dos listas

La concatenación de dos listas representadas como diferencias de listas se puede realizar de la siguiente manera:


concatenate(A - B, B - C, A - C).

Ventajas de la diferencia de listas

  • Permite realizar operaciones de concatenación y manipulación de listas de manera eficiente.
  • Evita la necesidad de recorrer toda la lista para agregar elementos al final.
  • Facilita la implementación de algoritmos que requieren manipulación de listas.

Ejemplo con qsort/2 utilizando append/3


qsort([], []).
qsort([H|T], Sorted) :-
    partition(>=(H), T, Less, Greater),
    qsort(Less, SortedLess),
    qsort(Greater, SortedGreater),
    append(SortedLess, [H|SortedGreater], Sorted).

Ejemplo con qsort/2 utilizando diferencia de listas


qsort([], L - L).
qsort([H|T], Sorted - Rest) :-
    partition(>=(H), T, Less, Greater),
    qsort(Less, Sorted - [H|SortedGreater]),
    qsort(Greater, SortedGreater - Rest).

Gramática de Cláusulas Definidas

Definite Clause Grammars

Las Definite Clause Grammars (DCG) son una extensión de Prolog que permite describir gramáticas formales de manera más natural y legible. Las DCG se utilizan comúnmente para el análisis sintáctico de lenguajes formales, como los lenguajes de programación y los lenguajes naturales. Las DCG se basan en la idea de que una gramática puede ser representada como un conjunto de cláusulas definidas en Prolog, donde cada cláusula representa una regla de producción de la gramática.

Ejemplo de DCG

Una gramática simple para reconocer cadenas formadas por 'a' y 'b' se puede definir utilizando DCG de la siguiente manera:


s --> [a], s, [b].
s --> [].

Explicación del Ejemplo

En este ejemplo, la regla 's --> [a], s, [b].' indica que una cadena válida puede consistir en una a, seguida de otra cadena válida (representada por s), seguida de una b. La regla 's --> []' indica que una cadena vacía también es válida. Por lo tanto, esta gramática reconoce cadenas como [a,b], [a,a,b,b], [a,a,a,b,b,b], etc.

Describiendo sequencias con seq//1, seq//2, y ...//0

Definimos un noterminal seq//1 para describir una secuencia de elementos:


seq([]) --> [].
seq([E|Es]) --> [E], seq(Es).

Ejemplo de uso:


?- phrase((`Hello, `,seq(Cs),`!`), `Hello, world!`),string_codes(C,Cs).
Cs = [119, 111, 114, 108, 100]
C = "world"

Describiendo sequencias con seq//1, seq//2, y ...//0

usando seq//1 podemos definir seq//2 para describir una secuencia de secuencias:


seq2([]) --> [].
seq2([Es|Ess]) --> seq(Es), seq2(Ess).

Ejemplo de uso:


?- phrase(seqq([`ab`, `cd`, `ef`]), Es), string_codes(S,Es)).
Es = [97, 98, 99, 100, 101,102]
S = "abcdef"

Describiendo sequencias con seq//1, seq//2, y ...//0

a veces es util describir "cualquier secuencia" y podemos hacerlo con el noterminal ...//0 que es equivalente a seq(_):


... --> [] | [_], ... .

Por ejemplo, el último elemento de una lista se puede describir con:


?- phrase((..., [Last]), `hello`), char_code(Char, Last).
Last = 111
Char = 'o'

Describiendo sequencias con seq//1, seq//2, y ...//0

Como otro ejemplo, elementos repetidos consecutivamente se pueden describir con:


?- phrase((..., [X,X], ...), `mississippi`), char_code(Char, X).
X = 115
Char = 's' ;
X = 115
Char = 's' ;
X = 112
Char = 'p' ;
false.

Inversión de listas, palíndromos y otros ejercicios.

DCG se pueden utilizar para describir una amplia variedad de estructuras de datos y patrones, como la inversión de listas, palíndromos, y otros ejercicios relacionados con cadenas y secuencias. Por ejemplo, para describir una lista que es un palíndromo (es decir, que se lee igual de izquierda a derecha que de derecha a izquierda), podemos definir la siguiente gramática:


palindrome --> [] .
palindrome --> [_] .
palindrome --> [X], palindrome, [X].

Inversión de listas, palíndromos y otros ejercicios.

Esta gramática reconoce cadenas como [], [a], [a,b,a], [a,b,b,a], etc.


?- phrase(palindrome, `neuquen`),
true. ;
false.

Inversión de listas, palíndromos y otros ejercicios.

También podemos describir la inversión de una lista utilizando DCG:


reversal([] --> [].
reversal([H|T]) --> reversal(T), [H].

Ejemplo de uso:


?- phrase(reversal(`abcd`, RevCodes), string_codes(Rev, RevCodes)).
RevCodes = [100, 99, 98, 97]
Rev = "dcba"

Inversión de listas, palíndromos y otros ejercicios.

DCG también se pueden utilizar para relacionar árboles binarios con sus recorridos en preorden, inorden y postorden. Por ejemplo, para describir el recorrido en inorden de un árbol binario, podemos definir la siguiente gramática:


tree_nodes(nil) --> [] .
          tree_nodes(node(Value, Left, Right)) --> 
          tree_nodes(Left),
          [Value],
          tree_nodes(Right).

Ejemplo:


?- phrase(tree_nodes(node(a, node(b, nil,
                     node(c, nil, nil)),
                     node(d, nil, nil))), Codes), string_codes(S, Codes)).
Codes = [98, 99, 97, 100]
S = "bcad"

Notación semicontextual

usar notación semicontextual podemos insertar elementos de la lista que inicialmente no estaban en la lista que se está analizando. Una regla DCG de la forma:


Head, [T1,...,Tn] --> Body.

puede leerse como: parsear la lista usando Body, luego agregar los terminos T1,...,Tn al resto de la lista. Por ejemplo:


nt1, [b] --> [a].
nt2      --> [b].

Notación semicontextual

El cuerpo de nt1//0 describe una lista con un unico elemento a. Operacionalmente, despues de que nt1//0 consume el atomo a, inserta el atomo b al frente del resto de la lista:


?- phrase((nt1, nt2), [a]).
true.

?- phrase(nt1, [a]).
false.

la version phrase/3 muestra lo que queda de la lista despues de que nt1//0 tiene éxito:


?- phrase(nt1, [a], Rest).
Rest = [b]

Notación semicontextual

Usando notación semicontextual , podemos implementar look-ahead para inspeccionar el siguiente elemento de la lista sin consumirlo:


look_ahead(T), [T] --> [T].

Ejemplo:


?- phrase(look_ahead(T), [a],Rest]).
T = a,
Rest = [a]

Pasando estados implícitamente

la notación semicontextual es util para el pasaje de estados implícita. Por ejemplo el siguiente contador de hojas de un árbol binario:


num_leaves(Tree, N) :-
            num_leaves_(Tree,0,N).

num_leaves_(nil, N0, N) :- N is N0 + 1.
num_leaves_(node(_,Left,Right), N0, N) :-
             num_leaves_(Left, N0, N1),
             num_leaves_(Right, N1, N).

Pasando estados implícitamente

siendo los estados una lista de un unico elemento entero:


num_leaves(Tree, N) :-
                phrase(num_leaves_(Tree), [0], [N]).

num_leaves_(nil), [N] --> [N0], { N is N0 + 1 }.
num_leaves_(node(_,Left,Right)) -->
              num_leaves_(Left),
              num_leaves_(Right).

Pasando estados implícitamente

Ejemplo de query:


?- num_leaves(node(a, node(b, nil, nil),
                 node(c, nil, 
                     node(d, nil, nil))), N).
N = 5.

los siguientes noterminales son utiles para describir estados con DCGs:


state(S), [S] --> [S].
state(S0, S), [S] --> [S0].

Pasando estados implícitamente

usando state//2 podemos escribir num_leaves_//1 de la siguiente manera:


num_leaves_(nil) --> state(N0, N), { N is N0 + 1 }.
num_leaves_(node(_,Left,Right)) -->
              num_leaves_(Left),
              num_leaves_(Right).
